# Assessments

Assessments (formerly called "Periods") represent evaluation cycles — a defined point in time when team members are evaluated on their skills.

## Creating an Assessment

Managers can create new assessments from any skillset view:

1. Navigate to a skillset
2. Click the **+** button next to the Assessment dropdown
3. Enter a name (e.g., "2026-Q1", "Annual Review 2026")
4. Optionally add a description
5. Click **Create**

::: tip Naming Convention
Use a consistent naming pattern like `YYYY-QN` (e.g., "2026-Q1") or descriptive names like "Mid-Year Review 2026" to keep assessments organized.
:::

## Selecting an Assessment

### In Skillset Views
The assessment dropdown in the header shows only assessments that have evaluation data for the current team/user context. Select one to view or edit evaluations for that assessment.

::: warning
The skillset view requires a specific assessment — there is no "All" option here, since evaluations and charts need a defined time context.
:::

### On the Dashboard
The Manager Dashboard includes an **Assessment** filter with an **All** option. When "All" is selected, stats aggregate across every assessment. Select a specific assessment to see stats for just that evaluation cycle.

## How Assessments Work

- Each evaluation record is linked to an assessment
- When you save evaluation scores, they are associated with the currently selected assessment
- Historical assessments are preserved — switching assessments lets you compare progress over time
- Assessments are shared across all skillsets and teams

## Migration from Periods

If your data was imported before the Assessments feature, existing period strings (like "2026-Q1") were automatically converted into Assessment records. No data was lost.
